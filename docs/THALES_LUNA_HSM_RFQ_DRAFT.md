### 1. Introduction
This RFQ is for the procurement of Thales Luna HSM units to support Phase 1 Scaling requirements for the Sovereign Unified Architecture. The project requires high-assurance cryptographic modules with strict compliance to the latest security standards.

### 2. Technical Specifications & Compliance Requirements
All proposed solutions must meet or exceed the following specifications:

#### 2.1 Security & Certification
*   **Certification:** FIPS 140-3 Level 3 Validated (MANDATORY).
*   **Environment:** Hardened, tamper-resistant network or PCIe appliance.
*   **Cryptographic Agility:** Support for Post-Quantum Cryptography (PQC) algorithms.

#### 2.2 Operational Features
*   **Firmware:** Must use NIST-inspected and validated firmware (e.g., v7.8.5 or v7.8.4).
*   **Cloning Protocol:** Support for CPv4 (Cloning Protocol version 4) for secure key replication.
*   **Partitioning:** Support for Per-Partition FIPS Mode, enabling independent FIPS-compliant and non-FIPS workloads.
*   **Algorithm Support:** Must block non-validated algorithms in FIPS mode (e.g., restricted 3DES usage).

### 3. Commercial Requirements
*   **Maintenance & Support:** 24/7 technical support with defined SLA response times.
*   **Delivery:** Global fulfillment capability to specified secure data centers.
*   **Transition Support:** Strategy for transitioning from FIPS 140-2 to 140-3 prior to the September 2026 historical status update.
